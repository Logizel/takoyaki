Features

Everything you'd check a dashboard for

Except you don't have to. It shows up where your team already talks.

Real-time pings

Get notified the moment someone pushes, opens a pull request, files an issue, or a CI/CD workflow completes. Each event comes with a link button straight to the relevant GitHub page. Private repos stay anonymous.

Contribution stats

Pull live data straight from GitHub: total counts, daily averages, and side-by-side comparisons with teammates.

Streak heatmap

Six months of commit history rendered as an ANSI heatmap, right inside Discord. No dashboard, no context switch.

Leaderboards

Rank the server by commits today or all year. Good-natured bragging rights, updated automatically.

Commands

Type less, ship the same

Eight slash commands cover everything from linking your account to checking who's on top.

/stats meShows your own contribution stats and current streak
/stats compareCompare your stats against someone else's
/stats topShows the yearly leaderboard for the server
/stats top-dayShows today's leaderboard for the server
/stats streakShows your 182-day grid streak as a heatmap
/github linkConnect your GitHub account to Discord
/github unlinkDisconnect your GitHub account
/setchannelSet the channel where Takoyaki posts activity notifications and configure Standard and Organisation mode
Setup

Three steps, then it just runs

From install to your first ping in under a minute.

01

Install the GitHub App

Add Takoyaki to your personal account or organization. For org tracking, install on the org directly.

02

Run /github link

Connect your Discord account to your GitHub username via OAuth.

03

Run /setchannel

Pick a channel and choose Standard Mode (personal pings) or Org Mode (anonymous org-wide events). Done.

Standard Mode: link your account and get pinged on your activity. Org Mode: track an entire organization without anyone needing to link accounts.

Privacy Policy

Takoyaki stores your GitHub username and OAuth access token. Nothing is shared with anyone else. They're used only to answer your own /stats requests.

Webhook events are processed in real time and then discarded. Only commit counts are kept, and only to power your streak.
